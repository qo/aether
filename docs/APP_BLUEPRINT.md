# App Blueprint

Purpose: define the user-facing Aether current product shape.

## What Is Confirmed / What Is Unknown

- [Confirmed in code] The UI has pages for Live Room, Experiment Console, Data Explorer, Knowledge Base, Agent Console, and System/Devices.
- [Unknown / needs verification] Final chart performance under high packet rates requires browser testing with real data.

## Pages

- Live Room: source badge, board status, session state, packet/RSSI/quality telemetry, RF field, waterfall, subcarrier bars, motion/occupancy trends, summary, event ticker, record controls.
- Experiment Console: protocol picker, start/stop, labels, notes, checklist, timeline.
- Data Explorer: session picker, comparison, raw metadata, derived features, quality metrics, export.
- Knowledge Base: searchable docs, research notes, outcomes, open questions, verification status.
- Agent Console: grounded chat-like interface with tool provenance.
- System/Devices: serial config, board statuses, firmware versions, health checks, calibration checklist.

## Visual Direction

Neutral enterprise instrument panel, semantic status colors, compact cards, and dense data-forward information design.
